Data method

The app keeps its boundary snapshot separate from current administrative information. Every data change must pass a recorded comparison and review.

1. Boundary snapshot

Production boundaries come from geoBoundaries Indonesia ADM2 boundary ID IDN-ADM2-22746128. ADM2 means second-level administrative boundary data. This source represents a 2020 snapshot. The app uses a simplified local file during normal startup and loads the detailed local file only for a detailed export.

2. Match administrative information

HDX tabular ADM2 information is added only when a normalized ADM2 name has one safe match. The Batch 1 result recorded 466 unique matches and 53 rows with more than one possible match. Those 53 shapes can still be colored, but the app does not force an official code, name, or province onto them.

3. Stable region IDs

canonical-provinces-v1.csv stores 38 current provinces. canonical-regions-v1.csv stores 519 stable region rows based on the boundary file. crosswalk-region-ids-v1.csv keeps older projects compatible. The filenames are retained because they are part of the data contract.

4. Simplified boundaries

The startup map is simplified from the detailed file with the Douglas-Peucker method at a tolerance of 0.018 degrees. Small island shapes are kept where practical, and automated checks cover several small-island examples.

5. Repeatable data and license checks

The offline data process checks the source inventory, license manifest, file checksums, data differences, and two independent rebuilds. The latest recorded result reproduced 10 comparable files with the same hash: a9f6c8612e79d6325dd9415d329851313637c4cf998b54d787f6811637cd13e8.

6. Spreadsheet data and region matches

Pasted data, CSV, TSV, and XLSX files use the same table rules. A verified region code is checked first, followed by the region name and province. More than one match, no match, duplicates, ignored rows, empty values, and values that are not valid remain separate statuses. The app never chooses an uncertain match silently.

7. Colors and exports

The app can color categories, equal ranges, ranked groups, manual ranges, or values around a midpoint. Only matched rows with valid values are included. An empty value is not zero. The map and export use the same legend. SVG, PNG, raster PDF, and match-report CSV files are created locally in the browser with the required boundary attribution.

Advanced boundary review

Batch 1 did not claim a legal-grade review of overlaps or gaps between shapes. A specialist review with native mapping software remains required before the boundary source is replaced.
