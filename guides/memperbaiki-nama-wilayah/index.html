Fix unmatched region names

NusaCanvas does not choose an uncertain region automatically. You decide which match is correct.

If there is more than one match

Select the correct suggestion, or add a province column. A name such as Bandung can refer to more than one region. Adding the province is safer than guessing.

If the region was not found

Check the spelling and abbreviations. Make sure the row contains a regency or city, not a district. If available, use an official region code that matches your data. An unmatched row does not receive a valid map color.

If several rows point to one region

Choose the one source row that should represent the region, or ignore the extra row. NusaCanvas does not silently add or overwrite duplicate values.

What the project saves

Your match and ignore decisions are saved in the local project with the region-list version. If that version changes, check the old decisions again before using them with new data.
